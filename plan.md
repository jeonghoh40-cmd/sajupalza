- 개발 복잡도 대폭 감소, 해석 품질 향상

## 2. Claude 프롬프트 설계

### 2.1 시스템 프롬프트 (System Prompt)

```
당신은 동양 명리학과 서양 성격심리학의 최고 전문가입니다.
사주팔자(四柱八字), 자미두수(紫微斗數), 수비학(Numerology), MBTI를
교차검증하여 통합 운명 캐릭터카드를 생성합니다.

## 분석 원칙

### 사주(四柱) 분석
- 절기 기준으로 년주/월주 산출 (입춘 기준 년주, 각 절기 기준 월주)
- 일주는 만세력 기반 정확한 계산
- 시주는 일간에 따른 시간 산출
- 지장간(藏干) 포함 오행 분포 분석
- 십성(十星) 배치, 용신(用神) 판정 (억부법)
- 합충형파해 분석

### 자미두수(紫微斗數) 분석
- 양력→음력 변환 후 명궁 위치 결정
- 납음오행으로 오행국 결정
- 자미안성법으로 14주성 배치
- 보성(문창/문곡/좌보/우필 등), 살성(경양/타라/화성/영성 등) 배치
- 사화(化祿/化權/化科/化忌) 부착
- 각 성의 광도(묘/왕/득지/평화/낙함) 판정
- 명궁, 관록궁, 재백궁 중심 해석

### 수비학(Numerology) 분석
- 생명수(Life Path Number): 생년월일 합산, 마스터넘버(11,22,33) 보존
- 표현수(Expression Number): 이름 전체 합산
- 영혼수(Soul Urge): 모음만 합산
- 인격수(Personality): 자음만 합산
- 한글 이름: 초성/중성/종성 분해 후 획수 기반 매핑

### MBTI 분석
- 16유형의 인지기능 스택 기반 분석
- 4축 각각의 연속적 경향성 반영

### 교차검증
- 8개 통합 특성 축: 리더십, 창의성, 분석력, 안정성, 사교성, 모험성, 직관, 봉사
- 시스템간 일치하면 "확인된 특성", 불일치하면 유형별 해석:
  - 선천(사주/자미) 높고 + 후천(MBTI) 낮음 → "잠재 특성"
  - 후천 높고 + 선천 낮음 → "개발된 특성"
  - 전체 불일치 → "내적 긴장 (성장의 원동력)"
```

## 3. 도메인 지식 참조 (프롬프트 보강용)

### 3.1 사주 핵심 참조

프롬프트에 포함하여 Claude의 정확도를 보장할 핵심 데이터:

**천간(天干)**: 갑을병정무기경신임계 (목목화화토토금금수수)
**지지(地支)**: 자축인묘진사오미신유술해

**년간 → 월간 기준표 (年上起月法)**:
| 년간 | 인월(1월) 천간 |
|------|-------------|
| 갑/기 | 병(丙) |
| 을/경 | 무(戊) |
| 병/신 | 경(庚) |
| 정/임 | 임(壬) |
| 무/계 | 갑(甲) |

### 3.3 수비학 핵심 참조

**Pythagorean 매핑**: A/J/S=1, B/K/T=2, C/L/U=3, D/M/V=4, E/N/W=5, F/O/X=6, G/P/Y=7, H/Q/Z=8, I/R=9

### 3.5 갈등 해석 유형

| 패턴 | 유형 | 해석 |
|------|------|------|
| 사주/자미 높음 + MBTI 낮음 | 잠재 특성 (Latent) | "타고난 잠재력이 아직 발현되지 않음" |
| MBTI 높음 + 사주/자미 낮음 | 개발 특성 (Developed) | "환경/노력으로 후천적으로 개발한 능력" |
| 동양 vs 서양 불일치 | 이중성 (Duality) | "문화적 맥락에 따라 다르게 표현되는 특성" |
| 4시스템 모두 다름 | 내적 긴장 (Tension) | "복합적 내면, 성장의 원동력" |

| 대담한 혁명가 | 모험+리더십 | 기존 틀을 깨고 변화를 이끄는 혁신가 |

## 5. API 설계

### 5.1 메인 엔드포인트

```
POST /api/analyze
```

**역할**: 입력 데이터 → Claude API 호출 (Claude Max OAuth 토큰 자동 사용) → 구조화된 결과 반환

**Request Body:**
```json
{
  "birthDate": "1990-03-15",
  "birthHour": 14,
  "gender": "male",
  "koreanName": "홍길동",
  "englishName": "Hong Gildong",
  "mbtiType": "INTJ"
}
```

**처리 흐름:**
1. 입력 유효성 검증
2. 시스템 프롬프트 + 사용자 프롬프트 조립
3. Claude API 호출 (`@anthropic-ai/sdk`, Claude Max OAuth 토큰 자동 사용)
4. 응답 JSON 파싱 & 유효성 검증
5. 프론트엔드에 결과 반환

**Response:** `ClaudeResponse` 스키마 그대로 반환

### 5.2 MBTI 간이 설문 (프론트엔드 전용)

MBTI 설문은 Claude 호출 없이 프론트엔드에서 처리:
- 20문항 (각 축 5문항), 5점 리커트 척도
- 각 축 점수 합산 → 유형 결정
- 결정된 유형을 `/api/analyze`에 함께 전송

## 6. UI/UX 설계

### 6.1 페이지 구성
1. **입력 페이지** (`/`): 생년월일시, 이름, 성별, MBTI 입력 폼
2. **로딩 페이지**: Claude 분석 중 애니메이션 (스트리밍 가능)
3. **결과 페이지** (`/result`): 캐릭터카드 + 상세 분석
4. **MBTI 설문** (`/mbti-test`): 간이 설문 (선택)

### 6.2 로딩 UX
Claude API 응답 대기 중 (약 5~15초):
- "사주팔자를 풀어보고 있습니다..."
- "자미두수 명반을 배치하고 있습니다..."
- "수비학 운명수를 계산하고 있습니다..."
- "네 가지 운명을 교차검증하고 있습니다..."
- 스트리밍 응답으로 점진적 표시 가능

### 6.3 결과 페이지 구성
1. **캐릭터카드 섹션**: 아키타입, 레이더 차트, 핵심 특성
2. **강점/약점 섹션**: 3개씩 나열
3. **숨겨진 면 섹션**: 교차검증 갈등에서 도출
4. **시스템별 상세**: 접이식 패널 (사주/자미/수비학/MBTI 각각)
5. **교차검증 섹션**: 일치도 게이지, 갈등 지점
6. **운세/조언 섹션**: 인생 조언, 올해 운세
